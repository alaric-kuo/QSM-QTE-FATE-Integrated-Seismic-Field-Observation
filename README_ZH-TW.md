# 1. 從這裡開始：本研究在觀測什麼

本研究提出一個直接的工程問題：

> **當地震運動進入結構之後，能勢會沿著哪些結構通道傳遞、在哪裡集中，而既有的感測紀錄能否讓我們看見這條持續演化的路徑？**

NEES 的既有實驗資料提供不同樓層的位移、速度與加速度歷時。這些量測通常被視為彼此分離的反應量；本版本則將它們共同讀取為一個隨時間演化的結構能勢狀態，並追蹤從地震能勢輸入、結構耦合、目標態命中、路徑集中，到後續位移反應的完整鏈條。

整體觀測可以濃縮為一行：

**地震能勢輸入 → 實測結構運動 → 演化中的能勢狀態 → 主導傳遞路徑 → 對可能需要介入位置的感知**

## 1.1 四組紀錄顯示了什麼

在四組地震紀錄與三種控制條件下，目前的實作反覆出現三項觀測。

第一，Kobe 0.35 半主動控制與 Morgan Hill 1.00 被動控制關閉案例，顯示 QSM 演化後的樓層 `a·v` 顯化，與下一時間步的實測反應具有高度一致性。兩者的平均絕對值包絡相關係數分別為 0.918 與 0.916。

第二，四組結構耦合案例皆反覆指向 `1F–2F` 樓層介面；相較之下，僅使用邊界輸入的參照案例，路徑權重仍接近均分。因此，內部路徑指示來自地震輸入與實測結構狀態的耦合，而非僅由地震輸入本身直接產生。

第三，El Centro 案例保留了混合訊號座標與衍生速度所造成的影響。這些不規則性沒有被清除，而是被保留在觀測紀錄之中，使資料來源、訊號處理方式與量測語意本身也成為分析的一部分。

目前的解析度停留在樓層域感知。構件層級損傷定位、實體能勢校準、自動警戒判準與閉迴路控制，仍屬後續實作階段。

# 3. 共鳴能勢梯度理論（Resonance Power Gradient Theory）的輸入語言

量子結構力學 QSM 理論階層由 共鳴能勢梯度 RPG 的質量生滅方程式開始：

$$
\frac{1}{m}\frac{dm}{dt}=\frac{a\cdot v}{c^2}
$$

局部單元功率判準為：

$$
P_u(t)=a(t)\cdot v(t)
$$

在工程尺度下，$`a\cdot v`$ 表示單位質量功率尺度：

$$
\frac{P}{m}=a\cdot v
$$

NEES 紀錄並未提供將每一樓層通道換算為絕對瓦特值所需的實體樓層質量，因此 V11 使用 $`a\cdot v`$ 作為經質量正規化、且與作功相容的單元功率輸入：

$$
\frac{dW}{m}\approx a\cdot du=a\cdot v\cdot dt
$$

程式同時保留帶正負號的 $`a\cdot v`$，以及其絕對值累積後、與作功相容的量。

# 4. 量子結構力學（Quantum Structural Mechanics）

## 4.1 從資產剛度到能勢通道

傳統結構力學寫成：

$$
F=Kx
$$

QSM 將剛度矩陣同時重新理解為抵抗資產與結構耦合圖。在第一個 QSM 通道條件中，對角線上的自體勢能井被拔除，使非對角耦合所形成的能勢流動通道得以顯影。

**QSM 本體論轉換：** 剛度矩陣 $`K`$ → 拔除對角線勢能井 → 凸顯非對角耦合 → 形成哈密頓能勢通道 → 載入地震能勢 → 波函數演化

通道傳遞係數 $`\gamma_{ij}`$ 描述能勢在節點 $`i`$ 與節點 $`j`$ 之間通過的能力。通道縮窄、損傷或阻塞，都會改變 $`\gamma_{ij}`$，並使演化中的結構狀態重新分流。

## 4.2 哈密頓能勢轉換算符

QSM 從標準哈密頓演化方程出發：

$$
i\hbar\frac{d}{dt}\lvert\Psi(t)\rangle=\hat H\lvert\Psi(t)\rangle
$$

QSM 定義 **哈密頓能勢轉換算符**：

$$
\hat H_p=-i\left(\frac{\hat H}{\hbar}\right)
$$

因此，能勢波函數狀態滿足：

$$
\frac{d}{dt}\lvert\Psi(t)\rangle=\hat H_p\lvert\Psi(t)\rangle
$$

其形式上的時間演化為：

$$
\lvert\Psi(t)\rangle=e^{\hat H_p t}\lvert\Psi(0)\rangle
$$

這個算符定義是 QSM 的核心：

- $`\hat H`$ 是結構通道法則的哈密頓描述。
- $`\hat H_p`$ 吸收相位因子 $`-i`$ 與尺度因子 $`\hbar^{-1}`$。
- $`e^{\hat H_p t}`$ 直接推進能勢波函數狀態。

## 4.3 QSM 核心能勢方程式與能勢顯化

QSM 的核心能勢方程式為：

$$
P(t)=(a\cdot v)\hat H_p\lvert\Psi(t)\rangle
$$

若寫成空間結構狀態：

$$
P(x,t)=(a\cdot v)\hat H_p\lvert\Psi(x,t)\rangle
$$

三個因子的角色彼此不同：

| QSM 物件 | 角色 |
|---|---|
| $`a\cdot v`$ | 真實動態燃料規模，也是單位質量功率尺度 |
| $`\hat H_p`$ | 能勢通過結構通道時的哈密頓能勢轉換 |
| $`\lvert\Psi(t)\rangle`$ | 持續演化的能勢波函數狀態 |

在實驗應用中，等效輸入也可寫成 $`P_{\mathrm{input}}(t)`$：

$$
P(t)\sim P_{\mathrm{input}}(t)\hat H_p\lvert\Psi(t)\rangle
$$

本版本以實測 $`a(t)\cdot v(t)`$ 作為 $`P_{\mathrm{input}}(t)`$ 的直接一階基礎。

## 4.4 保真度作為目標態命中比例

QSM 透過保真度觀測演化中的狀態命中了何處：

$$
Fid(t)=\left\lvert\langle\Psi_{\mathrm{target}}\vert\Psi(t)\rangle\right\rvert^2
$$

當目標被縮減為結構節點 $`n`$ 時：

$$
Fid^{(n)}(t)=\lvert\Psi_n(t)\rvert^2
$$

保真度是 QSM 的工程化觀測介面，可讀為目標態命中比例。真實的目標態命中能勢解讀為：

$$
P_{\mathrm{real}}^{\mathrm{target}}(t)\sim P_{\mathrm{input}}(t)Fid^{\mathrm{target}}(t)
$$

累積的目標命中等效作功為：

$$
W_{\mathrm{hit}}^{\mathrm{target}}(T)=\int_0^T P_{\mathrm{input}}(t)Fid^{\mathrm{target}}(t)\,dt
$$

這組方程將輸入尺度、演化狀態與目標命中三者清楚分開。

## 4.5 QSM 理論與 V11 程式碼的精確對應

V11 程式將等效通道矩陣存放在變數 `H` 中，接著計算：

```python
evals, evecs = np.linalg.eigh(H)
U = (evecs * np.exp(-1j * evals * dt)) @ evecs.conj().T
```

其數值形式為：

$$
U(\Delta t)=e^{-iH\Delta t}
$$

在正規化單位 $`\hbar=1`$ 下，對應：

$$
e^{\hat H_p\Delta t}=e^{-iH\Delta t}
$$

程式碼**沒有**把矩陣 `H` 重新定義成 $`\hat H_p`$。正確對應關係如下：

| 表示層次 | 物件 |
|---|---|
| 理論上的哈密頓通道法則 | $`\hat H`$ |
| QSM 哈密頓能勢轉換算符 | $`\hat H_p=-i(\hat H/\hbar)`$ |
| 程式層級的等效通道矩陣 | `H` |
| 程式層級的時間演化算子 | `U = exp(-iHΔt)` |

程式使用兩種等效通道矩陣：

```python
if operator_key == "laplacian":
    H = L
elif operator_key == "zero_diagonal":
    H = -W
```

零對角形式用來隔離非對角傳遞通道；拉普拉斯形式則承載 QTE 橋接所使用的純拓樸通道條件。

## 4.6 V11 的量測驅動狀態建構

理論上的 QSM 狀態為 $`\lvert\Psi(t)\rangle`$。既有實驗資料提供位移、速度與加速度紀錄，並未直接量測複數波函數。

因此，V11 將實測樓層訊號編譯為一個正規化複數狀態。程式使用：

```python
energy = v_norm**2 + (omega_norm * u_norm)**2 + 0.25 * a_norm**2
amp = np.sqrt(energy / np.sum(energy, axis=1, keepdims=True))
phase = np.arctan2(omega_norm * u_norm, v_norm)
psi = amp * np.exp(1j * phase)
```

這是為了讓本資料集進入 QSM 狀態空間而採用的 **V11 量測編碼**。它是一項實作選擇，不會取代 QSM 的典範方程，也沒有重新定義 $`\lvert\Psi(t)\rangle`$ 的理論意義。

## 4.7 單步演化與量測更新

在每一個時間步中，V11 使用截至時間 $`k`$ 的量測資料 → 建立當下複數結構狀態 → 以 `U = exp(-iHΔt)` 演化狀態 → 讀取演化後的目標態保真度 → 重建下一時間步的樓層 `a·v` 顯化 → 與 $`k+1`$ 時刻的實測 `a·v` 比較 → 同化 $`k+1`$ 的實測狀態。

程式層級的演化為：

```python
psi_prior = normalize_complex(U @ (psi + source_gain * dt * source))
```

完整狀態保真度與節點保真度計算為：

```python
state_fidelity = abs(np.vdot(psi_meas[k + 1], psi_prior)) ** 2
fid = np.abs(psi_prior) ** 2
```

用於單步比較的樓層等效作功顯化為：

```python
evolved_av_next[k + 1] = p_total * fid * evolved_sign
```

這一行是 V11 對下一時間步帶正負號 $`a\cdot v`$ 場的觀測性重建，也是目標態命中能勢的程式層級操作化。QSM 的核心能勢方程式仍為：

$$
P(t)=(a\cdot v)\hat H_p\lvert\Psi(t)\rangle
$$

V11 目前驗證的是狀態演化與保真度加權顯化鏈。完整算符作用後的能勢向量與絕對物理量校準，仍屬後續工程工作。

# 5. 量子拓樸快遞方法（Quantum Topology Express Method）

## 5.1 QTE 方法論主線

QTE 的方法論主線為：

**觀點 → 拓樸 → 通道 → 演化 → 顯影 → 行動**

其中，**顯影**是把演化過程轉為可讀的觀測量，使能勢密度、動態勢差、邊流、顯化速率、累積顯化量與顯化分數形成一條連續的判讀鏈。

其計算資料流為：

**$`R`$ → $`A`$ → $`W`$ → $`L_{\mathrm{geo}}`$ → $`V_{\mathrm{bg}}`$ → $`H`$ → $`\psi(0)`$ → $`\psi(t)`$ → $`\rho(t)`$ → $`\Delta V_{\mathrm{resp}}(t)`$ → $`J(t)`$ → $`\Gamma(t)`$ → $`m(t)`$ → $`M(t)`$**

其中：

| 符號 | QTE 意義 |
|---|---|
| $`R`$ | 節點集合 |
| $`A`$ | 鄰接矩陣 |
| $`W`$ | 幾何或通道權重 |
| $`L_{\mathrm{geo}}`$ | 幾何拉普拉斯 |
| $`V_{\mathrm{bg}}`$ | 背景能勢場 |
| $`H`$ | 完整的場驅動哈密頓量 |
| $`\psi(t)`$ | 演化中的 QTE 狀態 |
| $`\rho(t)`$ | 能勢密度 |
| $`\Delta V_{\mathrm{resp}}(t)`$ | 動態勢差 |
| $`J(t)`$ | 邊流 |
| $`\Gamma(t)`$ | 顯化速率 |
| $`m(t)`$ | 累積顯化量 |
| $`M(t)`$ | 顯化分數 |

## 5.2 從 QSM 橋接至 QTE

在 QTE 的純拓樸條件下：

$$
\hat H_p=\kappa L_{\mathrm{geo}}
$$

QTE 將背景勢能井重新放回系統，形成完整的場驅動哈密頓量：

$$
H=\hat H_p+\alpha_v\mathrm{diag}(V_{\mathrm{bg}})
$$

因此：

$$
H=\kappa L_{\mathrm{geo}}+\alpha_v\mathrm{diag}(V_{\mathrm{bg}})
$$

**QSM →** 先拔除勢能井，看能勢通道。  
**QTE →** 以 $`\hat H_p`$ 作為拓樸通道項，並加入背景能勢場 $`V_{\mathrm{bg}}`$，形成完整的場驅動哈密頓量。

完整的 QTE 狀態演化為：

$$
i\hbar\frac{d}{dt}\lvert\Psi(t)\rangle=H\lvert\Psi(t)\rangle
$$

並有：

$$
\lvert\Psi(t)\rangle=e^{-iHt/\hbar}\lvert\Psi(0)\rangle
$$

此處的 $`H`$ 是完整的 QTE 場驅動哈密頓量，同時包含通道項與背景勢能井。

## 5.3 QTE 可觀測量

能勢密度為：

$$
\rho_i(t)=\lvert\psi_i(t)\rvert^2
$$

動態勢差為：

$$
\Delta V_{\mathrm{resp}}(t)=-L_{\mathrm{geo}}\rho(t)
$$

邊流為：

$$
J_{ij}(t)=2\,\mathrm{Im}\left(\psi_i^*(t)H_{ij}\psi_j(t)\right)
$$

局部速度場由邊流導出：

$$
v_i(t)=\frac{\sum_j J_{ij}(t)u_{ij}}{\rho_i(t)+\epsilon}
$$

局部加速度為：

$$
a_i(t)\approx\frac{v_i(t+\Delta t)-v_i(t)}{\Delta t}
$$

局部單元功率與顯化速率為：

$$
P_{u,i}(t)=a_i(t)\cdot v_i(t)
$$

$$
\Gamma_i(t)=\frac{a_i(t)\cdot v_i(t)}{c^2}=\frac{P_{u,i}(t)}{c^2}=\frac{1}{m_i}\frac{dm_i}{dt}
$$

累積顯化量為：

$$
m_i(t)=m_i(0)\exp\left(\int_0^t\Gamma_i(\tau)\,d\tau\right)
$$

顯化分數整合 QTE 可觀測量與工程判準：

$$
M_i(t)=\lambda_1\lvert\Delta V_{\mathrm{resp},i}(t)\rvert+\lambda_2\max(\Gamma_i(t),0)+\lambda_3\max(\Delta V_{\mathrm{bg},i},0)+\lambda_4\log m_i(t)
$$

## 5.4 V11 已實作的樓層域 QTE

V11 在樓層域解析度下，已實作 QTE 資料流的前段與中段：

**$`R`$ → $`A`$ → $`W(t)`$ → $`L_{\mathrm{geo}}(t)`$ → 等效 $`H(t)`$ → $`\psi(t)`$ → 節點密度／保真度 → $`J_{12}(t),J_{23}(t)`$ → 動態樓層路徑顯影**

節點集合為：

```text
R = {1F, 2F, 3F}
```

加權鄰接矩陣為：

$$
W=
\begin{bmatrix}
0 & w_{12} & 0 \\
w_{12} & 0 & w_{23} \\
0 & w_{23} & 0
\end{bmatrix}
$$

度矩陣與拉普拉斯為：

$$
D_{ii}=\sum_j W_{ij}
$$

$$
L_{\mathrm{geo}}=D-W
$$

V11 的兩個觀測探針為：

| 探針 | 程式矩陣 | 理論解讀 |
|---|---|---|
| 零對角 | `H = -W` | QSM 非對角能勢通道觀測 |
| 拉普拉斯 | `H = Lgeo` | 在正規化 $`\kappa=1`$ 下的 QTE 純拓樸通道條件 |

目前尚未依材料容量、結構強度、損傷狀態、BIM 語意或感測器回傳的構件狀況，建立完整背景能勢場 $`V_{\mathrm{bg}}`$。因此，拉普拉斯探針代表的是 QTE 的**純拓樸條件**，尚非完整的場驅動哈密頓量。

## 5.5 邊流與路徑顯影

V11 計算：

$$
J_{12}(t)=2\,\mathrm{Im}\left(\psi_1^*(t)H_{12}\psi_2(t)\right)
$$

$$
J_{23}(t)=2\,\mathrm{Im}\left(\psi_2^*(t)H_{23}\psi_3(t)\right)
$$

路徑權重初始值為：

$$
w_{12}(0)=w_{23}(0)=1
$$

程式依據下列資訊更新兩條路徑權重：

- 邊流大小；
- 樓層間 $`a\cdot v`$ 差異；
- 量測殘差差異；
- 可選用的位移反應回饋；
- 路徑記憶衰減。

兩條路徑權重正規化為：

$$
w_{12}(t)+w_{23}(t)=2
$$

樓層域路徑主導指標為：

$$
D_p(t)=\frac{w_{12}(t)-w_{23}(t)}{w_{12}(t)+w_{23}(t)}
$$

解讀如下：

| 指標 | 樓層域解讀 |
|---|---|
| $`D_p>0`$ | `1F–2F` 路徑顯影較強 |
| $`D_p<0`$ | `2F–3F` 路徑顯影較強 |
| $`\lvert D_p\rvert\le 0.02`$ | 最終路徑權重接近均分 |

這套自適應路徑規則，是 V11 的樓層域觀測機制，尚未等同於完整的 QTE 鏈條 $`\Gamma(t)`$、$`m(t)`$ 與 $`M(t)`$。

# 6. 碎形生機拓樸演化論（Fractal Alive Topology Evolution）

## 6.1 核心方程

FATE 為：

$$
\mathrm{F.A.T.E.}=
\mathrm{Aware}_{\mathrm{power}}
\cdot
\mathrm{Alert}_{\mathrm{control}}
\cdot
\mathrm{Alive}_{\mathrm{evolve}}
$$

**中文方法論方程式：** 感知能勢波動路徑 → 警戒控制致命拓樸 → 存活演化開放耗散

三個階段與 QTE 的關係如下：

| FATE 階段 | 與 QTE 的關係 | 工程角色 |
|---|---|---|
| `Aware_power` | 觀點 → 拓樸 → 通道 → 演化 → 顯影 | 感知能勢波動路徑，讀取能勢匯入、通道流動、集中、反應與資料來源 |
| `Alert_control` | 顯影 → 行動 | 警戒控制致命拓樸，並形成工程控制輸出 |
| `Alive_evolve` | 行動 → 再演化 | 存活演化開放耗散；改寫 $`V_{\mathrm{bg}}`$、$`W`$、$`L_{\mathrm{geo}}`$、$`\hat H_p`$ 或 $`H`$，並觀測新的狀態 |

FATE 介入可以改變阻尼、剛度、耦合強度、反力或能勢傳遞路徑。被改寫後的場會進入下一輪演化循環，使局部顯化成為可承受的代謝，而非主體崩塌。

## 6.2 V11 中的 `Aware_power`

V11 透過下列內容建立 `Aware_power` 觀測紀錄：

1. **入射波感知**  
   邊界輸入單獨探針，用來觀察邊界來源經樓層拓樸後可以產生什麼。

2. **結構耦合狀態感知**  
   樓層狀態同化，將實測內部結構狀態放入 QSM 演化循環。

3. **目標態感知**  
   以保真度記錄演化中的能勢波函數狀態命中了何處。

4. **拓樸路徑感知**  
   動態 $`w_{12}`$、$`w_{23}`$、$`D_p`$、$`J_{12}`$ 與 $`J_{23}`$ 保留樓層域路徑的演化歷程。

5. **作功與反應感知**  
   將目標命中作功與位移側的等效作功紀錄進行比較。

6. **資料溯源感知**  
   直接量測與衍生訊號、座標差異、數值微分、資料擷取與處理方式，都保留在每一案例紀錄中。

7. **消融條件感知**  
   樓層狀態、僅邊界輸入、固定路徑、零對角、拉普拉斯與反應回饋條件，用來分離 QSM、QTE 與目前反應項的不同作用。

# 7. 生命週期部署：設計、竣工與營運

QSM、QTE 與 FATE 在建築生命週期中使用同一條理論鏈；隨著證據條件不同，內容會從設計假設，發展為經驗證的實體性質，再進入即時回饋。因此，方法的部署依序從**探索可能的能勢通道**，發展到**依實際建築資料建構場驅動通道**，最後進入**營運階段對演化場進行修正與控制**。

| 生命週期階段 | 可使用的證據 | QSM–QTE–FATE 的角色 |
|---|---|---|
| **設計** | 幾何、拓樸、邊界條件、設計材料、候選結構系統與可能地震輸入 | 探索地震能勢如何與幾何及拓樸關係互動；編譯 $`R`$、$`A`$、$`W`$、$`L_{\mathrm{geo}}`$，並在施工前比較通道連續性、集中、阻塞、弱面配置與耗散方案。 |
| **竣工／試運轉** | 實際幾何、已安裝構件斷面、材料性質、接頭、連接、阻尼器、隔震裝置、測試結果與經確認的邊界條件 | 將設計拓樸轉換為由實體場驅動的通道模型。實際材料與容量資訊形成背景能勢場 $`V_{\mathrm{bg}}`$，使拓樸通道項與背景勢能井共同進入完整的場驅動哈密頓量。 |
| **營運** | 感測量測、位移、速度、加速度、應變、層間位移、裝置狀態、檢查紀錄、維護歷史與可觀測損傷 | 持續更新 $`\Psi(t)`$、$`V_{\mathrm{bg}}(t)`$、$`W(t)`$ 與 $`H(t)`$；FATE 透過 `Aware_power` 讀取演化場，透過 `Alert_control` 評估介入，並將修改後的系統送回 `Alive_evolve`。 |

## 7.1 設計階段的拓樸與地震能勢輸入探索

設計階段的主要任務，是觀察可能的地震能勢輸入如何遭遇預定的幾何與拓樸：

**候選地震能勢輸入 → 幾何與語意關係 → $`R`$ → $`A`$ → $`W`$ → $`L_{\mathrm{geo}}`$ → 可能能勢通道 → 集中、阻塞、反射與耗散路徑**

在此階段，QSM 使結構耦合所承載的能勢通道顯影；QTE 則將不同觀點、拓樸與通道配置所造成的演化差異顯影出來。這些比較可支援結構配置、弱面治理、阻尼器配置、隔震策略與可接受局部耗散路徑的決策。

## 7.2 竣工後的場驅動通道

建築完成後，模型可以導入實際交付的物理建築。經確認的材料性質、構件尺寸、接頭狀況、已安裝裝置與試運轉結果，共同提供背景能勢場：

$$
H=\hat H_p+\alpha_v\mathrm{diag}(V_{\mathrm{bg}})
$$

或以 QTE 通道形式表示：

$$
H=\kappa L_{\mathrm{geo}}+\alpha_v\mathrm{diag}(V_{\mathrm{bg}})
$$

竣工階段因此將**實際材料容量**與**實際拓樸**連接起來，建立一個可供後續感測與結構變化比較的初始場驅動哈密頓量。

## 7.3 營運期間的感測、修正與主動控制

營運期間，感測資料會把結構當下狀態送回模型：

**感測器回傳 → 狀態與場修正 → $`\Psi(t)`$、$`V_{\mathrm{bg}}(t)`$、$`W(t)`$、$`H(t)`$ → 更新後的 QSM–QTE 演化 → FATE 感知、警戒、介入與再演化**

主動或半主動介入可以改變阻尼、剛度、耦合強度、反力或能勢傳遞路徑。系統被改變之後，再次進入觀測，形成完整 FATE 循環：

$$
\mathrm{Aware}_{\mathrm{power}}
\rightarrow
\mathrm{Alert}_{\mathrm{control}}
\rightarrow
\mathrm{Alive}_{\mathrm{evolve}}
$$

目前的 NEES 版本位於**營運階段觀測**的位置：利用既有感測紀錄，重建樓層尺度的狀態演化與拓樸路徑顯影。設計階段的 BIM／幾何編譯、竣工階段的材料場建構，以及即時閉迴路控制，構成下一階段的實作方向。

# 8. V11 端到端計算流程

```text
1. 讀取原始 NEES 資料檔。

2. 選取或推導樓層位移 u(t)、速度 v(t) 與加速度 a(t)。

3. 建立實測單元功率紀錄：
   Pu(t) = a(t)·v(t)。

4. 將 u、v、a 編譯成 V11 複數量測狀態。

5. 建立目前樓層域通道矩陣：
   零對角 H = -W
   或
   拉普拉斯 H = Lgeo。

6. 計算：
   U = exp(-iHΔt)。

7. 將目前狀態向前演化一個時間步。

8. 計算：
   完整狀態保真度
   與
   節點保真度 |ψi|²。

9. 重建下一時間步的樓層帶正負號 a·v 顯化。

10. 將 k+1|k 的演化結果
    與 k+1 時刻的實測 a·v 比較。

11. 同化新的實測狀態。

12. 計算邊流 J12 與 J23。

13. 對動態路徑探針更新 w12 與 w23。

14. 累積目標命中作功與下游反應證據。

15. 儲存完整歷程、圖表、摘要、資料溯源與執行紀錄。
```

# 11. 跨案例發現

## 11.1 QSM 單步狀態演化

具有直接速度通道的 Kobe 與 Morgan Hill 案例，在演化後樓層 $`a\cdot v`$ 顯化與下一時間步實測紀錄之間，呈現最強的帶正負號相關與絕對值包絡相關。

| 案例 | 平均帶正負號相關係數 | 平均絕對值包絡相關係數 |
|---|---:|---:|
| Kobe 0.35 半主動控制 | 0.656 | 0.918 |
| Morgan Hill 1.00 被動控制關閉 | 0.649 | 0.916 |

El Centro 案例的帶正負號一致性較低，但仍保留相當程度的絕對值包絡對應。這項結果與原始通道中混合座標及衍生速度的資料來源一致。

## 11.2 邊界輸入與內部結構狀態

在四組紀錄中，僅使用邊界輸入的參照案例，最終路徑權重皆接近均分；加入樓層狀態同化後，則反覆形成內部下層介面指示。

這項差異顯示，目前設定下所觀測到的樓層路徑，取決於結構耦合狀態，無法僅由邊界輸入單獨恢復。

## 11.3 反覆出現的樓層域路徑指示

四組樓層狀態案例皆指向 `1F–2F` 樓層介面，但各案例在集中、轉換、恢復與再分配歷程上仍彼此不同。

這個共同介面是一項重複出現的樓層域觀測。若要進一步做元件層級的物理解讀，仍需要原始結構配置、控制系統配置、感測器位置與構件層級拓樸。

## 11.4 邊流

樓層狀態動態探針顯示，`1F–2F` 的均方根邊流集中程度高於 `2F–3F`。邊流比提供一項具有相位敏感性的觀測，與最終路徑權重形成互補。

## 11.5 路徑顯影與位移反應

主導樓層路徑與最大位移反應樓層，屬於不同的觀測層次：

| 觀測層 | 意義 |
|---|---|
| QTE 路徑顯影 | 演化中的能勢狀態在哪裡集中或通過 |
| 下游位移反應 | 運動在哪裡轉化為可見幾何變形 |

Repository 同時保留兩種歷程。

## 11.6 證據分離

固定路徑探針與動態路徑探針，保留了相近的單步絕對值包絡相關係數，因此，單步 $`a\cdot v`$ 對齊主要可歸於 QSM 狀態演化。

動態路徑權重與邊流集中，則構成 QTE 的路徑顯影證據。

目前的 FATE 證據，是持續更新的 `Aware_power` 紀錄。控制與再演化階段仍為下一步實作。

# 12. 目前證據與發展邊界

本四案例版本，建立了一套共同的 QSM 狀態演化、QTE 樓層域路徑觀測與 FATE `Aware_power` 計算紀錄。最強的證據包含：Kobe 與 Morgan Hill 直接通道紀錄中的重複單步對齊、僅邊界輸入與結構耦合觀測之間的分離，以及四組案例皆反覆出現的 `1F–2F` 樓層介面指示。

|---|---|---|
| **QSM** | 哈密頓能勢狀態演化、目標態保真度、與作功相容的目標命中重建 | $`\hat H`$、$`\hat H_p`$、樓層質量與絕對能勢單位的實體校準；長時間自主演化；構件層級目標態 |

# 17. 研究定位

在同一套且未變更的計算設定下，V11 將 QSM 哈密頓能勢狀態演化、目標態保真度、QTE 樓層域路徑顯影與 FATE `Aware_power`，應用於四組既有地震實驗紀錄。具有直接速度通道的 Kobe 與 Morgan Hill 案例，重現了高度的單步 QSM 對齊；四組結構耦合案例皆指向 `1F–2F` 介面，同時保留不同事件各自的形成、集中、轉換、恢復與再分配歷程；El Centro 案例則揭露資料來源與座標語意如何影響 $`a\cdot v`$ 的一致性。

本版本建立了一條可重現的實驗橋梁：從 **QSM 能勢狀態演化**，通過 **QTE 拓樸路徑顯影**，進入 **FATE 感知能勢波動路徑**。放入建築生命週期後，同一條理論鏈可以始於設計階段對地震能勢與幾何拓樸的探索，發展為利用實際材料資料建立竣工場驅動哈密頓量，最後進入營運階段的感測修正、回饋與主動控制系統。
